# Учебный Блокчейн: От Нуля до Распределённой Сети

> Пошаговое руководство по созданию собственного блокчейна с P2P-сетью на реальном оборудовании

---

## Введение

Этот проект — практическое погружение в технологию блокчейн. Мы пройдём путь от простейшей цепочки блоков до полноценной распределённой сети с майнингом, транзакциями и криптографической защитой.

## Часть 1. Теоретический фундамент

### 1.1. История: Сатоши Накамото и Виталик Бутерин

#### Сатоши Накамото (2008–2009)

Сатоши Накамото — псевдоним неизвестного автора (или группы авторов), который в октябре 2008 года опубликовал whitepaper *"Bitcoin: A Peer-to-Peer Electronic Cash System"*. 

**Проблема, которую решал Сатоши:**
- Как передавать ценность напрямую между людьми без банков и посредников?
- Как предотвратить двойную трату (double spending) без центрального арбитра?
- Как достичь консенсуса в распределённой системе, где узлы не доверяют друг другу?

**Решение Bitcoin:**
- Распределённый реестр (леджер) транзакций
- Proof-of-Work консенсус — вычислительная работа как доказательство честности
- Криптографическая связка блоков через хеши
- Экономические стимулы для майнеров

#### Виталик Бутерин (2013–2015)

19-летний программист из Торонто (родом из России) задался вопросом: а что если блокчейн — это не только про деньги?

**Идея Ethereum:**
- Блокчейн как глобальный компьютер
- Смарт-контракты — программы, исполняемые на всех узлах сети
- Тьюринг-полный язык программирования (Solidity)
- Децентрализованные приложения (dApps)

**Ключевое отличие:**
- Bitcoin: "Отправить X монет с адреса A на адрес B"
- Ethereum: "Выполнить произвольный код при выполнении условий"

### 1.2. Что под капотом блокчейна

#### Структура блока

```
┌─────────────────────────────────────┐
│              БЛОК #N                │
├─────────────────────────────────────┤
│  previous_hash    │ хеш блока N-1   │
│  timestamp        │ время создания  │
│  nonce            │ число для PoW   │
│  merkle_root      │ корень дерева   │
│  transactions[]   │ список транзакц │
├─────────────────────────────────────┤
│  hash = SHA256(all_above)           │
└─────────────────────────────────────┘
            │
            ▼
┌─────────────────────────────────────┐
│              БЛОК #N+1              │
│  previous_hash = hash блока N       │
│  ...                                │
└─────────────────────────────────────┘
```

#### Хеширование (SHA-256)

Хеш-функция преобразует данные любого размера в строку фиксированной длины (256 бит для SHA-256). 

Свойства:
- **Детерминированность** — одинаковый вход всегда даёт одинаковый выход
- **Необратимость** — по хешу нельзя восстановить исходные данные
- **Лавинный эффект** — изменение 1 бита меняет ~50% битов хеша
- **Устойчивость к коллизиям** — практически невозможно найти два разных входа с одинаковым хешем

#### Proof-of-Work (доказательство работы)

```
Задача: найти nonce такой, что
SHA256(block_data + nonce) начинается с N нулей

Например, при difficulty=4:
hash должен начинаться с "0000..."

Единственный способ найти — перебор (brute force)
```

Это и есть "майнинг" — вычислительная работа, которую легко проверить, но сложно выполнить.

#### Консенсус

Как узлы договариваются, какая цепочка "правильная"?

**Правило longest chain:**
- Каждый узел принимает самую длинную (с наибольшей накопленной работой) цепочку
- При форке — работаем над своей веткой, но переключаемся, если чужая стала длиннее
- Атака 51% — контроль большинства хешрейта позволяет переписывать историю

## Часть 3. Этапы разработки

Проект разбит на 6 уровней сложности. Каждый уровень — работающий блокчейн, который расширяется на следующем этапе.

### Уровень 1: Минимальный блокчейн (1 файл, ~100 строк)

**Цель:** Понять базовую структуру блокчейна.

**Что реализуем:**
- Класс Block с хешированием
- Класс Blockchain с валидацией цепочки
- Простой Proof-of-Work
- Генезис-блок

**Запуск:**
```bash
python level1_basic.py
```

**Результат:** Локальная цепочка блоков с майнингом в консоли.

---

### Уровень 2: Транзакции и кошельки (~200 строк)

**Цель:** Добавить экономику.

**Что добавляем:**
- ECDSA-ключи (приватный/публичный)
- Адреса кошельков (хеш публичного ключа)
- Подписанные транзакции
- UTXO или Account-based модель
- Coinbase-транзакции (награда майнеру)

**Запуск:**
```bash
python level2_transactions.py
```

**Результат:** Можно создавать кошельки и отправлять монеты.

---

### Уровень 3: REST API (~300 строк)

**Цель:** Взаимодействие через HTTP.

**Что добавляем:**
- FastAPI-сервер
- Эндпоинты:
  - `GET /chain` — получить цепочку
  - `GET /blocks/{index}` — получить блок
  - `POST /transactions` — отправить транзакцию
  - `GET /wallet/new` — создать кошелёк
  - `GET /wallet/{address}/balance` — баланс
  - `POST /mine` — запустить майнинг
- Swagger-документация (автоматически)

**Запуск:**
```bash
uvicorn level3_api:app --host 0.0.0.0 --port 5000
```

**Результат:** Блокчейн доступен по HTTP, можно тестировать через браузер или curl.

---

### Уровень 4: P2P-сеть (~400 строк)

**Цель:** Распределённая сеть узлов.

**Что добавляем:**
- WebSocket-соединения между нодами
- Протокол сообщений:
  - `HANDSHAKE` — представление ноды
  - `GET_CHAIN` / `CHAIN` — синхронизация
  - `NEW_BLOCK` — широковещание нового блока
  - `NEW_TX` — широковещание транзакции
  - `GET_PEERS` / `PEERS` — обмен списками пиров
- Хранение списка известных пиров
- Автореконнект при разрыве

**Запуск:**
```bash
# Нода 1
python level4_p2p.py --port 5000

# Нода 2
python level4_p2p.py --port 5001 --peers localhost:5000
```

**Результат:** Несколько нод синхронизируют цепочку.

---

### Уровень 5: Консенсус и форки (~500 строк)

**Цель:** Обработка конфликтов.

**Что добавляем:**
- Longest chain rule
- Обработка форков
- Реорганизация цепи при получении более длинной
- Orphan-блоки (блоки без родителя)
- Mempool (пул неподтверждённых транзакций)
- Валидация блоков от других нод

**Тестовый сценарий:**
```bash
# Изолируем ноду, майним блоки
# Подключаем обратно — наблюдаем реорганизацию
```

**Результат:** Сеть устойчива к форкам и партиционированию.

---

### Уровень 6: Production-ready (~700+ строк)

**Цель:** Готовность к реальному использованию.

**Что добавляем:**
- **Seed-нода** — точка входа для новых узлов
- **Автообнаружение** — mDNS для локальной сети
- **Режимы работы:**
  - `--seed` — bootstrap-нода
  - `--full` — полная нода с майнингом
  - `--light` — только кошелёк
- **Персистентность** — сохранение цепи в SQLite
- **CLI-интерфейс** — удобное управление
- **Метрики** — количество пиров, хешрейт, высота цепи
- **Graceful shutdown** — корректное завершение
- **Логирование** — структурированные логи

## Часть 7. Упражнения и эксперименты

### Эксперимент 1: Атака 51%

1. Запустите 3 ноды с майнингом
2. Отключите одну ноду от сети
3. На изолированной ноде намайните 10 блоков
4. Подключите обратно
5. Наблюдайте реорганизацию

### Эксперимент 2: Двойная трата

1. Создайте транзакцию A→B на 100 монет
2. Одновременно создайте A→C на те же 100 монет
3. Отправьте в разные ноды
4. Посмотрите, какая попадёт в блок

### Эксперимент 3: Сложность майнинга

1. Измените difficulty с 4 на 5
2. Измерьте время майнинга блока
3. Постройте график зависимости

### Эксперимент 4: Партиционирование сети

1. Разделите сеть на две группы
2. Майните в обеих группах
3. Объедините — какая цепь победит?
